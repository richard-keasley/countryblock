# Country Block

Block access to website according to country 

## Stack
- **OS:** Linux
- **Web Server:** Apache
- **Database:** MySQL (optional, for advanced features)
- **Language:** PHP 7.4+

## Architecture

Geo-blocking is applied at the application level through a PHP entry point that checks the visitor's country before routing to any page content. All requests are routed through `index.php` via Apache's `.htaccess` rewrite rules.

### Geolocation Data Source

Use **MaxMind GeoLite2** for IP-to-country mapping:

- **Free tier:** GeoLite2 (requires free account registration at [maxmind.com](https://www.maxmind.com/en/geolite2/geolite2-free-geolocation-databases))
- **Includes:** Country, city, ASN databases; updated weekly
- **Formats:** Binary (.mmdb) or CSV
- **Self-hosted:** Download and store locally; no external API calls needed
- **Alternative (API-based):** MaxMind GeoIP2 API for real-time lookups (requires API key)

Other options:
- **ip2location.com** - Paid; reliable ASN and proxy detection
- **ipstack.io** - API-based; free tier available but limited
- **GeoNames** - Free country/city data; requires manual IP matching

**Recommendation:** Use MaxMind GeoLite2 binary database locally for best performance—no external API calls, low latency.

## How it's organized

```
index.php                  Entry point; geo-blocking middleware
.htaccess                  Apache rewrite rules, security headers
composer.json              PHP dependencies (MaxMind GeoIP2)

src/
  CountryBlocker.php       Core geo-blocking class

config/
  allowed_countries.php    Whitelist of ISO 3166-1 alpha-2 country codes

public/
  home.php                 Example home page
  api.php                  Example API endpoint

scripts/
  download_geoip_db.php    Script to download/update MaxMind database

data/
  GeoLite2-Country.mmdb    MaxMind GeoIP database (generated by script)

tests/                     Unit tests (PHPUnit)
```

**How it fits together:** 

1. Apache receives a request and rewrites it to `index.php` (unless it's a real file/directory)
2. `index.php` loads the `CountryBlocker` class
3. `CountryBlocker` extracts the visitor's IP from request headers (supporting Cloudflare, proxies, etc.)
4. The class queries the MaxMind GeoLite2 database to determine the visitor's country
5. If the country is in the whitelist (`config/allowed_countries.php`), request continues
6. If blocked, the visitor receives a 403 Forbidden response
7. Allowed requests are routed to the appropriate page (`public/home.php`, etc.)

## How to Run

### Prerequisites
- PHP 7.4+ with curl extension
- Apache with `mod_rewrite` enabled
- Composer package manager

### 1. Clone and install dependencies
```bash
git clone https://github.com/richard-keasley/countryblock.git
cd countryblock
composer install
```

### 2. Register for free MaxMind GeoLite2 account
- Visit https://www.maxmind.com/en/geolite2/geolite2-free-geolocation-databases
- Create a free account and generate a license key
- Note your Account ID

### 3. Download GeoIP database
```bash
export MAXMIND_ACCOUNT_ID=your_account_id
export MAXMIND_LICENSE_KEY=your_license_key
php scripts/download_geoip_db.php
```

Or use composer script:
```bash
composer run download-geoip
```

### 4. Configure allowed countries
Edit `config/allowed_countries.php`:
```php
<?php
return [
    'US', // United States
    'GB', // United Kingdom
    'CA', // Canada
];
```

### 5. Deploy to Apache

Copy all files to your Apache DocumentRoot:
```bash
sudo cp -r /path/to/countryblock /var/www/html/
sudo chown -R www-data:www-data /var/www/html/countryblock
sudo chmod -R 755 /var/www/html/countryblock
sudo chmod 777 /var/www/html/countryblock/data  # For database updates
```

### 6. Enable Apache rewrite module
```bash
sudo a2enmod rewrite
sudo systemctl restart apache2
```

### 7. Test

Visit http://localhost/ in a browser. If your country is allowed, you'll see the welcome page. Otherwise, you'll get a 403 error.

Test with a spoofed IP:
```bash
curl -H "CF-Connecting-IP: 8.8.8.8" http://localhost/
```

## Configuration

### Allowed Countries
Edit `config/allowed_countries.php` to add/remove countries:
```php
<?php
return [
    'US', // United States
    'GB', // United Kingdom
    'CA', // Canada
    'AU', // Australia
    'FR', // France
];
```

Find country codes at: https://en.wikipedia.org/wiki/ISO_3166-1_alpha-2

### Custom Block Message
In `index.php`, modify the `block()` call:
```php
$blocker->block("Custom message shown to blocked users");
```

### Update Database Periodically
MaxMind updates the GeoLite2 database weekly. Set up a cron job:
```bash
0 0 * * 0 cd /var/www/html/countryblock && php scripts/download_geoip_db.php
```

## Testing

Run unit tests:
```bash
composer test
```

Test geo-blocking manually:
```bash
# Test allowed country (US)
curl -H "CF-Connecting-IP: 1.1.1.1" http://localhost/ -i

# Test blocked country
curl -H "CF-Connecting-IP: 185.220.101.1" http://localhost/ -i
```

## Security Considerations

- `.htaccess` denies direct access to `config/` and `vendor/` directories
- Database file is not web-accessible
- IP validation prevents header injection
- Supports Cloudflare and proxy X-Forwarded-For headers
- HTTPS recommended for production
